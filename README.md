# SCM Mini Project: Student Login Module
This repository contains the SCM Mini Project deliverables, demonstrating professional Software Configuration Management practices using **Next.js**, **Prisma**, and **MUI**.

## GitHub Link
- https://github.com/Abel-Tessema/scm

## Project Overview

The goal of this project was to implement a Student Login Module while maintaining strict version control, change management, and configuration auditing protocols.

## Tech Stack

- **Framework:** Next.js (App Router)
- **UI Library:** Material UI (MUI)
- **Database ORM:** Prisma
- **Language:** TypeScript

## Repository Structure

Aligned with the SCMP, the repository is organized as follows:
- `/src`: Application source code.
- `/docs`: All SCM documentation (Plans, Baselines, CRs, Audit).
- `/tests`: (Optional) Path for testing files.
- `/releases`: (Optional) Path for build artifacts.
- `/prisma`: Database schema and configurations.

## SCM Process and Milestones

1. **Baselines**
   - **Baseline 1 (BL1):** Established the governance layer, including the SCM Plan, Requirements, and CI Register.
   - **Baseline 2 (BL2):** Established the functional layer, including the working prototype and implementation of three approved Change Requests (CRs).<br><br>

2. **Change Control**<br>
We simulated a real-world Configuration Control Board (CCB) process:
   - CR-01: Branding update (Login button color change).
   - CR-02: Schema enhancement (Module description field).
   - CR-03: Legal compliance (Copyright footer addition).<br><br>

3. **Release History**
   - **v1.0:** Initial functional release with basic Login UI and Database models.
   - **v1.1:** Final release incorporating all merged CRs and updated documentation.
